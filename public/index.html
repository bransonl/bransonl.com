Bulletin

Bulletin is a group-based bulletin board web-app. The idea is for multiple users to have shared bulletin boards where each user can create new, edit existing or delete posts on the board. These posts are then displayed, grouped and coloured based on user created groups and with size depending on the importance of the post set by the user.

The front end is built with AngularJS using D3.js Treemap to visualize the posts. The back end uses Node.js with Express and a PostgreSQL database to store account and board info.

This project is currently in progress.

2

2 is a real-time multiplayer Big 2 web-app capable of handling multiple simultaneous games in separate rooms. It also has a built in chat so players can speak to one another during a game. Also, it has spectator functionality, allowing people to watch games if the room is full, the game is in progress or if they just don't want to play. There is also a player substitution feature where players can swap in a spectator to play for them.

The server provides a variety of commands depending on the person's role of host, player or spectator, with most restricted to the host of the room. Information on these commands can be retrieved using /help in the chat box.

The front end is built with AngularJS. The back end uses Node.js with Express and Socket.io to handle websockets and allow simultaneous real-time connections separated by room. The game can be found at bransonl.com/2.

Lumberhack

alisonlee.co

alisonlee.co is a website developed for a graphic designer to showcase their portfolio of work. As many graphic designers do not have the desire to learn HTML, CSS and JavaScript to develop and maintain their own websites, I developed a system using which the graphic designer can easily add to, update or remove their work from the website, much like a CMS but significantly simpler. Also, all file modifications are done by the designer on the server where the files are hosted.

The system depends on a certain organization of files on the server, a file to determine the selection and order of work shown, and files to determine the order in which the content for each piece of work is displayed in the additional writeup for each piece. Using these files, the client-side JavaScript systematically retrieves the corresponding files from the server and places them onto the DOM in the order specified by the designer.

The portfolio section of the website acts similarly to a SPA. The "single page" aspects are implemented with only DOM manipulation using JavaScript with the JQuery library since a larger framework such as AngularJS would be highly unneccessary for the amount of content and the features of the website.
